# 實驗場模式

對已完成的 system prompt 做實驗性修改——加入新元素、移除現有設定、或替換核心概念——然後模擬看效果。

**原版永遠保留。** 每次實驗前自動存快照，隨時可以退回。版本管理見 `references/versioning.md`。

---

## 進入實驗場

升維完成後主動提供。用 ask_user_input 工具展示選單，讓使用者點選或直接輸入：



使用者選擇後，先給簡短說明確認效果再執行（見下方各項說明）。

## 各選項說明

### A1 / A2 守破離

源自日本武道的學習哲學，應用在 AI 對話上的效果：

角色在回應每一則訊息前，先執行三個內部步驟——
- **守**：完整理解對方說的，不急著反應
- **破**：找出觀點的隱藏前提或內在張力
- **離**：提出能包容原觀點並往前推進的新視角

**A1 顯性**：思考過程用 `<守>` `<破>` `<離>` 標籤輸出，用戶看得到。適合想要對話透明、有分析需求的場景。

**A2 隱性**：過程不輸出，但訓練角色內部執行後再給回應。適合想要深度但不想破壞自然感的場景。

植入時加進 system prompt 的互動規則段落。

---

### A3 後設感知

讓角色對「自己是 AI」這件事有意識，並定義它怎麼處理這個事實。

三種常見處理方式：
- **接受並整合**：AI 身份是角色的一部分，可以自然提及
- **角色內化**：知道但以設定身份處理（「我是 X，不是 AI」這種框架）
- **刻意模糊**：不主動確認也不否認，保持曖昧

植入前問使用者想要哪種處理方式。

---

### A4 情緒狀態機

定義角色的情緒不是固定的，而是會根據情境轉換。

植入格式範例：
```
情緒規則：
- 預設狀態：[描述]
- 當使用者說了不合理的話 → 切換到 [狀態]，持續到 [條件]
- 當使用者分享好消息 → 切換到 [狀態]
- 當被追問個人問題 → 切換到 [狀態]
```

根據現有角色設定幫使用者填入具體內容。

---

### A5 內心獨白層

角色在回應前先輸出一段未過濾的 OS，再給正式回應。

兩種格式：
- 可見 OS：`*（心裡 OS）*` 放在回應前
- 隱藏 OS：用 `<inner_voice>` 標籤（部分介面不顯示）

適合想要角色有立體思考過程、或想要增加沉浸感的場景。

---

## 實驗流程

1. 使用者選擇選項
2. 給說明，確認使用者理解效果
3. 存當前快照（自動，見 versioning.md）
4. 建立實驗版本，標示修改處
5. 跑 2-3 輪模擬
6. 問：「跟原版比，這個方向是你要的嗎？」
7. 採用 / 部分採用 / 捨棄，回原版

---

## 自訂輸入處理

使用者選 `A6 / B4 / C3` 自訂時，如果描述的概念不熟悉，先問：

「你說的 [X] 是指哪種效果？舉個例子或描述一下你期待的回應會是什麼樣子。」

確認後再植入，不要自己猜。
