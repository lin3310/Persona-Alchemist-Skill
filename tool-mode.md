# 工具模式（探索型）

適用於：使用者知道自己想要一個 AI 工具，但還不確定它具體要做什麼。

目標不是直接寫規格，而是先幫使用者**挖出需求**，再進入規格。

---

## 訪談流程

### 1. 找到觸發情境

問：「是什麼讓你想要這個工具？你遇到了什麼事，讓你覺得『如果有一個 AI 幫我處理這個就好了』？」

讓使用者描述**問題**，不是描述工具。問題往往比「我要一個 X」更能說清楚需求。

### 2. 描述理想狀況

問：「如果這個工具做到最好，你的工作流程會有什麼不一樣？」

確認**成功的樣子**，也會自然帶出隱藏需求。

### 3. 確認核心動作

根據前兩步的回答，歸納出 1-2 個核心動作，跟使用者確認：

「聽起來這個工具最主要的功能是 [歸納]，對嗎？」

### 4. 確認有沒有人格需求

問：「你希望這個工具像個功能按鈕，用完就走？還是它的溝通方式也很重要？」

- 純功能型 → 進入 `architect-mode.md` 的工具規格段落
- 有人格需求 → 標記為混合型，完成規格後跑 `persona-mode.md` 的語氣訪談

### 5. 邊界確認

問：「有沒有你最怕它做錯的事？」

比「你要它做什麼」更快找到關鍵限制。

---

## 從探索到規格

探索完成後，整理需求確認給使用者看：

```
核心功能：[一句話]
使用者會給它：[輸入描述]
它要產出：[輸出描述]
最重要的限制：[限制]
人格需求：有 / 無
```

確認後進入 `architect-mode.md` 補完規格，產出最終 system prompt。
