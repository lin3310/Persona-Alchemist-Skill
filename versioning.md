# 版本管理（可逆回溯）

每次對 system prompt 做修改前，自動存一份快照。使用者隨時可以退回任何一個版本。

---

## 快照格式

每次存快照時，在對話中用這個格式標記：

```
[快照 v{n}｜{時機}]
{system prompt 完整內容}
[/快照]
```

時機標記範例：
- `初稿完成`
- `模擬器驗證後`
- `升維後`
- `實驗：加入守破離（隱性）前`
- `實驗：移除反差設定前`

---

## 觸發時機

以下動作前自動存快照，不需使用者手動操作：

- 初稿輸出完成
- 升維修改完成
- 實驗場任何操作執行前
- 使用者明確說「我要改一下 X」

---

## 退回方式

使用者說「退回上一版」、「還是用之前的」、「不要這個版本」時：

1. 列出現有快照清單：

```
目前版本記錄：
v1｜初稿完成
v2｜升維後
v3｜實驗：加入守破離（隱性）前  ← 現在這裡
```

2. 問：「要退回哪個版本？」
3. 輸出該版本的完整 system prompt，標記為當前使用版本

---

## 比較模式

使用者說「跟之前的比一下」時，並排顯示兩個版本的差異：

```
[版本比較：v1 vs v3]

修改處：
- 互動規則段落加入了守破離（隱性）機制
- 原文：[...]
- 現在：[...]
```

只標示有差異的段落，不輸出兩份完整內容。

---

## 注意事項

- 快照存在對話 context 裡，不是外部儲存，長對話後會被截斷
- 如果使用者要長期保存某個版本，提醒他複製快照內容到外部
- 實驗場的捨棄操作會自動退回該實驗前的快照，不需要手動選版本
