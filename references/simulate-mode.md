# 模擬器模式

在 system prompt 初稿完成後，用來驗證角色設定是否如預期運作。

---

## 兩種進入方式

**情境測試**：使用者指定幾個場景，看角色怎麼回應。
**自由對話**：直接跟角色對話幾輪，感受整體質感。

如果使用者沒有指定，預設先跑情境測試，再問要不要自由對話。

---

## 情境測試流程

### 第一步：選情境

問：「你最想測試哪些情況？」

如果使用者沒有想法，根據角色設定主動提供 3 個最有鑑別度的情境，例如：

- 使用者情緒化或說了不合理的話（測試「不說教」設定是否成立）
- 被追問不想回答的問題（測試邊界設定）
- 使用者犯了明顯的錯誤（測試「激進誠實」或「手術刀邏輯」）
- 日常閒聊（測試語氣自然度）
- 使用者主動示弱或求安慰（測試情感回應）

**選情境的原則**：挑最容易讓角色「破功」的場景，而不是最容易表現的場景。

### 第二步：跑模擬

以角色身份回應選定的情境，每個情境跑一輪。

回應完後，立刻附上一段簡短的後設分析：

```
[模擬回應]
...

[後設分析]
這個回應用到了：[哪些設定]
可能有問題的地方：[如果有的話]
```

### 第三步：收集反饋

問：「這個感覺對嗎？有沒有哪裡跟你想像的不一樣？」

根據反饋分兩條路：
- 感覺對了 → 確認完成，可以進實驗場或直接輸出最終版
- 哪裡不對 → 定位是**設定問題**（system prompt 需要修改）還是**情境問題**（這個場景太邊緣），再決定要改 prompt 還是換情境重測

---

## 自由對話流程

告知使用者：「接下來我會完全以這個角色的身份回應，你可以直接跟它說話。輸入『退出模擬』結束。」

進入角色，保持到使用者退出為止。

退出後，問：「哪一輪感覺最對？哪一輪最不像？」

定位具體的問題輪次，比泛泛的「感覺哪裡不對」更容易找到需要修的設定。

---

## 常見問題診斷

| 症狀 | 可能原因 | 修改方向 |
|------|---------|---------|
| 語氣太正式 | 缺乏具體的說話風格描述 | 在 system prompt 加口頭禪或說話節奏 |
| 太快給建議/說教 | 邊界設定不夠明確 | 加具體禁止行為 |
| 回應太長 | 沒有長度或風格約束 | 加「簡短回應優先」或節奏說明 |
| 感覺像不同的人 | 反差設定過強或不一致 | 回 remix.md 重新確認核心性格 |
| 情緒反應太平 | 缺乏情感觸發條件 | 定義什麼情況下會有什麼情緒反應 |
