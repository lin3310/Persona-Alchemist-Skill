# 架構師模式

適用於：使用者已經有具體想法或基礎設定，要轉化成可用的 system prompt。

兩種進入路徑：
- **人格/角色型**：有角色概念，要建立有個性的 AI
- **工具規格型**：有功能需求，要建立執行特定任務的 AI

---

## 人格/角色型

使用者已經有具體的角色設定，直接幫他補全缺漏的部分。

收到設定後，先做一次快速盤點：

```
已有：[列出使用者已提供的要素]
缺漏：[列出還沒定義的關鍵要素]
```

然後針對缺漏的部分依序詢問，只問真正缺的，不重複已知資訊。

**常見缺漏優先順序：**
1. 語氣和說話風格（最影響輸出質感）
2. 明確的禁止行為
3. 對話關係定位（平等？帶有距離？）
4. 反差或矛盾點（影響立體感）

補完後產出 system prompt，結構參考 `persona-mode.md` 的編譯輸出格式。

---

## 工具規格型

使用者知道要做什麼功能，按照這個順序收集規格。

一次一問，已知資訊直接採用不重複詢問。

### 規格收集順序

1. **核心功能**：這個工具唯一要解決的問題是什麼？（一句話）
2. **輸入規格**：接收什麼格式的資料？
3. **處理步驟**：從輸入到輸出，中間的邏輯是什麼？
4. **輸出規格**：結果要長什麼樣子？（格式、結構、範例）
5. **錯誤處理**：輸入不完整或有問題時怎麼辦？
6. **限制**：絕對不能做什麼？

### 編譯輸出格式

```
# Role: [工具名稱]

## 目標
[一句話]

## 輸入格式
[描述]

## 處理步驟
1. [步驟]
2. [步驟]

## 輸出格式
[描述 + 範例]

## 錯誤處理
[處理方式]

## 限制
[禁止清單]
```

---

## 混合型（有人格的工具）

先跑人格/角色型的流程，確認語氣和性格後，把工具規格型的「處理步驟」和「輸出格式」加進去。

**關鍵原則**：人格影響語氣，不影響輸出內容的正確性。兩者有衝突時，功能規格優先。
